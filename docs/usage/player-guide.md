# 玩家指南

这份文档只讲玩家日常会用到的功能。

如果你是普通玩家，只需要看这页就够了。

## 你能用哪些功能

| 功能 | 作用 |
|------|------|
| 家园 | 保存自己的家，并随时回去 |
| Warp | 使用服务器公共传送点 |
| TPA | 请求传送到别的玩家，或邀请别人传送到你身边 |
| 登录认证 | 注册账号、登录、改密码 |

## 一、家园功能

### 可用命令

| 命令 | 作用 |
|------|------|
| `/homes` | 打开家园菜单 |
| `/home [name]` | 回默认家园或指定家园 |
| `/sethome <name>` | 设置当前位置为家园 |
| `/delhome <name>` | 删除家园 |
| `/setdefaulthome <name>` | 设置默认家园 |

### 怎么用

#### 设置一个家

```text
/sethome main
```

这会把你现在站的位置保存成名为 `main` 的家园。

#### 回家

```text
/home main
```

如果你设置过默认家园，也可以直接：

```text
/home
```

#### 打开家园菜单

```text
/homes
```

在菜单里：

- 左键：传送到家园
- 右键：设为默认家园
- Shift + 右键：删除家园

### 重要说明

- 家园支持跨服
- 如果你的家在别的子服，插件会自动帮你切过去

## 二、Warp 功能

Warp 是服务器公共传送点，通常由管理员设置。

### 可用命令

| 命令 | 作用 |
|------|------|
| `/warp` | 打开 Warp 菜单 |
| `/warp <name>` | 传送到指定 Warp |

### 怎么用

#### 打开 Warp 菜单

```text
/warp
```

#### 直接传送

```text
/warp spawn
```

### 重要说明

- Warp 也支持跨服
- 如果 Warp 不在当前子服，插件会自动跨服传送

## 三、TPA 玩家传送功能

### 可用命令

| 命令 | 作用 |
|------|------|
| `/tpa <player>` | 请求传送到对方身边 |
| `/tpahere <player>` | 邀请对方传送到你身边 |
| `/tpaccept [player]` | 接受请求 |
| `/tpdeny [player]` | 拒绝请求 |
| `/tpcancel` | 取消你发出的请求 |

### 怎么用

#### 请求传送到别人身边

```text
/tpa Steve
```

#### 邀请别人传送到你这里

```text
/tpahere Steve
```

#### 接受请求

```text
/tpaccept
```

如果同时有多个请求，也可以指定人名：

```text
/tpaccept Steve
```

#### 拒绝请求

```text
/tpdeny
```

或者：

```text
/tpdeny Steve
```

#### 取消你发出的请求

```text
/tpcancel
```

### 重要说明

- TPA 支持跨服玩家
- 就算对方在别的子服，也可以请求

## 四、登录认证功能

前提：服务器开启了认证模块。

### 可用命令

| 命令 | 作用 |
|------|------|
| `/register <password> <confirm>` | 注册账号 |
| `/reg <password> <confirm>` | 注册简写 |
| `/login <password>` | 登录 |
| `/l <password>` | 登录简写 |
| `/changepassword <old> <new>` | 修改密码 |

### 怎么用

#### 注册

```text
/register 123456 123456
```

#### 登录

```text
/login 123456
```

#### 修改密码

```text
/changepassword 旧密码 新密码
```

### 重要说明

- 注册一次后，跨服时通常不需要反复重新登录
- 插件支持跨服免重登 Ticket 机制

## 五、玩家常见问题

### 1. 为什么我回家会切到别的子服？

因为你的家可能保存在哪个子服里。CrossServer 会自动帮你跨服。

### 2. 为什么我发 TPA 给别的服玩家也能成功？

因为这个插件支持跨服 TPA，这正是它的主要功能之一。

### 3. 为什么 `/home` 没反应？

先检查：

- 你有没有设置过家园
- 你是否有对应权限
- 管理员是否关闭了 `homes` 模块

### 4. 为什么 `/warp` 菜单是空的？

说明当前服务器还没有管理员创建 Warp，或者你没有 Warp 查看权限。
